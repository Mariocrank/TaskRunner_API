Modelado del sistema

En esta carpeta se encuentran los diagramas utilizados para representar la estructura y el diseño del sistema desarrollado en el proyecto.

El objetivo del modelado es facilitar la comprensión de cómo están organizadas las entidades y cómo se relacionan entre sí dentro de la aplicación.

Diagramas incluidos
Diagrama Entidad-Relación (DER)

El archivo DER.png representa las entidades principales del sistema y las relaciones que existen entre ellas.
Este diagrama permite visualizar la estructura de los datos y sirve como base para el diseño de la base de datos.

Diagrama UML

El archivo UML.png muestra la estructura general del sistema a nivel de clases, incluyendo sus atributos y relaciones.
Este diagrama ayuda a entender la organización del código y la lógica del proyecto.

Importancia del modelado

El modelado permite:

Tener una visión clara del sistema antes y durante el desarrollo

Evitar errores en el diseño de la base de datos

Facilitar el mantenimiento y futuras mejoras del proyecto
